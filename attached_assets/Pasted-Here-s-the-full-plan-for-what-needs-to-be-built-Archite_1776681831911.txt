## Architecture Plan

### Backend Changes

**1. New `User` model** (separate from `Member` — these are staff/admins, not library members)
```
User {
  name, email, password (bcrypt hashed),
  role: "admin" | "librarian",
  createdBy: ObjectId,  ← who added this user
  isActive: boolean
}
```

**2. New routes/controllers**
```
POST /api/auth/login        → returns JWT token
POST /api/auth/logout
GET  /api/auth/me           → get current user from token

GET  /api/users             → admin only: list all staff users
POST /api/users             → admin only: create new staff user
PUT  /api/users/:id         → admin only: edit user
DELETE /api/users/:id       → admin only: deactivate user
```

**3. Auth middleware**
```ts
// Two middleware functions:
authenticate   → verifies JWT, attaches user to req
authorizeAdmin → blocks non-admins (librarians can use app but can't manage users)
```

**4. Protect all existing routes** — wrap every route with `authenticate`

---

### Frontend Changes

**5. Sign In page** (`/login`)
- Email + password form
- Stores JWT in `localStorage`
- Redirects to Dashboard on success

**6. Protected Route wrapper**
```tsx
// Redirects to /login if no token
<ProtectedRoute> → wraps all existing pages
```

**7. New Users Management page** (`/users`) — admin only
- Table of all staff accounts
- Add new librarian/admin modal (admin only sees this)
- Deactivate user button

**8. Layout updates**
- Show logged-in user's real name + role in sidebar footer
- Logout button
- Hide "Users" nav item for librarian role
